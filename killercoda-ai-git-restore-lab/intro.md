# AI支援ツールにより退避されたGit管理を復旧せよ

あなたは VSCode と Cline を使って、ある小さなコードベースを修正していました。

ところが、修正後に `git status` や `git commit` を実行しようとすると、なぜか Git 管理が効かなくなりました。
調べると、ワークスペースの隠しファイルとして `.git_disabled` が残っており、同階層にあるはずの `.git` が消えています。

どうやら AI 支援ツールの安全機能が誤作動し、Git の管理情報を退避したまま戻していないようです。
この演習では、ワークスペースを元に戻して `git status` が使える状態へ復旧します。

## 目標

- ワークスペースの状態を確認する
- `.git_disabled` が何を意味するか見抜く
- Git 管理を復旧する
- `git status` が正常に動くことを確認する

環境の準備は自動で進みます。終わったら右のターミナルで `/root/lab/workspace` を確認してください。
